# 📱 安卓计时器应用 - AI开发提示词

```markdown
# 项目目标
请帮我开发一款安卓平台的计时器应用，用于记录工作、学习、运动等活动的耗时。

# 核心要求
- 界面清爽干净、操作简便
- **后台计时必须持续运行**（应用切换到后台或锁屏时不能停止）
- 适配安卓15系统

# 技术栈
- 语言：Kotlin
- 最低SDK：Android 11 (API 30)
- 目标SDK：Android 15 (API 35)
- UI：Jetpack Compose
- 架构：MVVM
- 数据库：Room

---

# 功能需求

## 1. 计时功能（核心）
| 操作 | 说明 |
|------|------|
| 开始 | 点击后开始计时 |
| 暂停 | 点击后暂停计时，按钮变为"恢复" |
| 恢复 | 点击后继续计时，按钮变为"暂停" |
| 停止 | 终止计时，保存记录到历史 |

- 显示格式：HH:mm:ss（秒级精度）
- 应用切换到后台/锁屏时，计时必须继续
- 通知栏显示当前计时状态，支持暂停/恢复/停止操作
- 应用重启后能恢复之前的计时状态

## 2. 分组管理
- 可创建、编辑、删除分组（如工作、学习、运动等）
- 每条计时记录关联一个分组
- 可按分组筛选历史记录

## 3. 历史记录
保存每条记录的以下信息：
- 计时时长（x小时y分钟z秒）
- 开始日期和时间
- 结束日期和时间
- 所属分组
- 备注（可选）

支持功能：
- 列表查看（按时间倒序）
- 按日期、分组筛选
- 删除单条记录

---

# 界面要求
- 采用 Material Design 3 风格
- 支持深色/浅色主题
- 主色调：清爽蓝色或绿色系
- 计时数字大字号、清晰易读

# 主要页面
1. 首页：计时显示 + 操作按钮 + 分组选择
2. 分组管理页：分组列表 + 增删改功能
3. 历史记录页：记录列表 + 筛选功能

---

# 后台计时实现要求
- 使用前台服务（Foreground Service）确保计时不被系统杀死
- 通知栏持续显示计时状态
- 使用时间戳计算时长（避免累加误差）
- 定期保存计时状态到数据库，支持应用重启后恢复

---

# 交付内容
1. 完整可编译的源代码
2. AndroidManifest.xml（含必要权限配置）
3. README使用说明
4. 关键代码注释

---

# 测试重点
请确保以下场景测试通过：
- [ ] 应用切换后台后计时继续
- [ ] 锁屏后计时继续
- [ ] 通知栏可操作暂停/恢复/停止
- [ ] 应用重启后能恢复计时状态
- [ ] 长时间运行无内存泄漏
```
